Hvad dælen er Git?

Git er noget vi skal bruge fordi det åbenbart er nemt? Not! Anyways. Her er nogle commandoer!

Git cd "folder"

Man skal huske at "connecte" sin Git mappe så det "Repository" man vil hente, ender det rigtige sted.

Det gør man først ved at åbne sin Comman Promt og skrive "cd (slip din git folder i vinduet)"

Så har Git selv fundet ud af hvor den skal smide alle de filer du henter.

Git Clone

Den første gang man henter en fil, skal man clone det

Det at clone en fil, vil sige at man henter en fil via en URL

Inde i commando prompt skriver man: git clone

Git status

Git status viser noget status på dine filer.

Hvad der er push, commit eller add'et.

Git status kan bruges hele tiden, det er bare en status kommando.

Git add .

Man bruger "Git add ." når man skal have gemt ens process. Punktumet betyder at man gemmer alle filer i folderen

Man skal altid bruge "Git add" før man bruger de andre kommandoer ift. at dele sin fil.

Git Commit -m "message"

Man bruger "git commit -m "message" til at fortælle andre hvilke ændringer/tilføjelser eller ting man har slettet samt hvem der har gjort det og hvornår

Man kan sige, at når man "comitter" tager man et "screenshot" af filen som man har ændret og gemmer det i databasen

Git push

Man bruger git push til at sende det man tilføjer til sit projekt online, så alle kan tilgå det.

Det er vigtigt at man ikke git pusher for meget, fordi man kan ende med at overright andres versioner af det samme projekt.

Hold derfor øje med din gruppes versioner, og "git commit -m" hvis man bare gerne vil gemme.

Git pull

Med git pull henter man de filer der allerede er arbejdet på.

Git help

Med git help får man en oversigt over kommandoerne, hvis man nu ikke lige kunne huske dem.
